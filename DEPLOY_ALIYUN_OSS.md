# 阿里云 OSS 部署说明

这个项目是 Vite + React 纯前端项目，部署到阿里云 OSS 时只需要上传 `dist` 目录里的静态文件。

## 1. 本地构建

双击运行：

```text
build-for-aliyun-oss.bat
```

或手动执行：

```bash
npm.cmd install
npm.cmd run build
```

构建完成后会生成：

```text
dist/
```

上传时要上传 `dist` 里面的所有文件，而不是上传 `dist` 文件夹本身。

## 2. 创建 OSS Bucket

建议：

- 地域：华东 1（杭州）、华东 2（上海）、华南 1（深圳）等中国内地区域
- 读写权限：公共读
- 存储类型：标准存储
- Bucket 用途：只放这个网站的静态文件

注意：如果 Bucket 位于中国内地，并绑定自定义域名，需要域名完成 ICP 备案。

## 3. 上传文件

进入 OSS 控制台：

```text
Bucket -> 文件管理 -> 文件列表
```

把 `dist` 目录里的内容全部上传到 Bucket 根目录。

上传后 Bucket 根目录应类似：

```text
index.html
assets/
```

## 4. 开启静态网站托管

进入：

```text
Bucket -> 数据管理 -> 静态页面
```

配置：

- 默认首页：`index.html`
- 子目录首页：不开通
- 默认 404 页：`index.html`
- 错误文档响应码：`200`

这是 React 单页应用的推荐配置，刷新页面或访问不存在路径时也会回到 `index.html`。

## 5. 绑定自定义域名

建议使用自己的域名，例如：

```text
plan.example.com
```

在 OSS 中绑定自定义域名，并到 DNS 解析处添加 CNAME 到 OSS 提供的域名。

重要：不要长期依赖 OSS 默认 Bucket 域名访问 HTML。阿里云文档说明，直接使用 Bucket 域名访问 HTML 可能被浏览器下载而不是在线预览；正式网站建议绑定自定义域名。

## 6. HTTPS

绑定域名后，在阿里云给该域名配置 HTTPS 证书。

## 7. 数据说明

当前项目数据保存在浏览器 `localStorage`。

上线后：

- 本地 `127.0.0.1:5173` 的数据不会自动出现在新域名里。
- 新域名会有独立的一份浏览器本地数据。
- 可以先在旧页面使用“站内本地备份”，再考虑后续增加云端同步。

如果你需要手机、不同电脑之间同步任务，需要额外接入云端数据库或后端接口；OSS 只能托管静态页面。
